[h1]S.A.M Framework — Support All Mods[/h1]

Mod Barony with plain JSON files. No C++, no compiler, no engine knowledge. Write a .json, put it in your mods folder, and the game loads it as a real class, item, monster or spell. Add a short script next to it if you want it to actually do something.

[b]S.A.M is a dependency, not a playable mod.[/b] On its own it adds nothing to play — it's the thing other mods are built on. Install it once and any S.A.M mod works.

[hr][/hr]

[h1]What you can make[/h1]
[list]
[*][b]Classes[/b] — attributes, skills, starting gear, starting spells, level-up growth, starting gold, a character-select portrait, and a forced head/look per race.
[*][b]Items and weapons[/b] — new items in Barony's own item table, with category, equip slot, weight, value, ATK/AC, model and icon.
[*][b]Monsters[/b] — variants of any creature with your own stats, gear, inventory, name, followers and spawn locations.
[*][b]Spells[/b] — custom spells that really cast, using the game's own projectile pipeline.
[*][b]Vanilla patches[/b] — edit the game's own data files in layers, so two mods stack instead of overwriting each other.
[*][b]Custom .vox models[/b] — your own voxel models, loaded at startup.
[/list]

[hr][/hr]

[h1]Scripting — Lua, JavaScript, TypeScript[/h1]
JSON says what your content [i]is[/i]. A script says what it [i]does[/i]. Drop a .lua, .js or .ts file next to any class or item JSON and it loads automatically (TypeScript is transpiled for you). No C++, no compiler.

[b]57 hooks[/b] — combat (on_hit, on_kill, on_block, on_before_damage), lifecycle (level up, death, floor change, game start), items (including picking one up off the ground), shops, chests, traps, monsters, keyboard input, a per-tick handler, world interactions (doors, levers, fountains, sinks, orb pedestals, teleporters, projectile impacts), and bound actions (Attack / Defend / Use) that follow whatever the player rebound them to.

[b]53 host functions[/b] — grant items and gold, read and set stats, apply effects, cast spells, deal or modify damage, set move speed, read equipped items, timers, persistent per-mod data, full monster scripting by UID, and runtime patching of existing content that reverts when your mod unloads.

Two functions ever get called: [b]on_event(event)[/b] and [b]on_tick(event)[/b]. Everything else arrives as [b]event.name[/b] inside on_event.

[b]Safe by design[/b] — scripts run sandboxed with a memory cap and an instruction budget. A broken script disables itself and logs why; it can't crash your game. Save backups are made automatically while mods are active, and no vanilla file is ever changed on disk.

[hr][/hr]

[h1]How to install[/h1]
S.A.M is a modified [b]barony.exe[/b], because reading all this needs engine changes. Steam can't replace a game's .exe, so you install it yourself. It takes a minute.
[list]
[*][b]1.[/b] Own Barony on Steam and launch it once so the game files exist.
[*][b]2.[/b] Download the installer below and run it. It finds Barony, backs up your barony.exe as [b]barony_vanilla.exe[/b], and installs the S.A.M one.
[*][b]3.[/b] Start Barony. If [b]sam_log.txt[/b] appears next to the exe, S.A.M is running.
[*][b]4.[/b] Put any S.A.M mod in [b]Barony/mods/[/b] and turn it on in the Mods menu.
[/list]
To go back to normal Barony, put barony_vanilla.exe back.

[h1]How to make a mod[/h1]
The easiest way is the browser tool. Nothing to download:

[b][url=https://spider12223.github.io/SAM-Framework/]➤ S.A.M Mod Builder[/url][/b]

Build a class, item, monster or spell with dropdowns and sliders, watch the JSON update live, then Export a ready-to-use .zip. It checks your work against the same schemas the game uses, so if it exports, it loads. You can also import an existing mod to edit it, or write straight into your Barony/mods folder in one click.

Writing JSON by hand works too — every file has a schema, so point your editor at it and get autocomplete:
[b][url=https://spider12223.github.io/SAM-Framework/docs/schema-reference.html]➤ Schema reference[/url][/b]

[hr][/hr]

[h1]Publishing your own mod[/h1]
A mod is just a folder: [b]mod.json[/b] plus your [b]classes/[/b], [b]items/[/b] and [b]monsters/[/b] JSON.
[list]
[*][b]1.[/b] Build and test it locally with S.A.M installed.
[*][b]2.[/b] Upload the folder as a Workshop item. Put a [b]preview.png[/b] in the folder — that exact filename is what gets used as the thumbnail, and without it your item has none.
[*][b]3.[/b] Add [b]S.A.M Framework[/b] as a Required Item on your Workshop page.
[*][b]4.[/b] Say in your description that players need S.A.M installed, and link this page. Subscribing alone won't install the exe.
[/list]
Mods built on S.A.M stack cleanly — layered patches and namespaced ids mean they don't clobber each other, and S.A.M checks that everyone in a multiplayer lobby is running the same set of mods.
