[h1]S.A.M Framework (Support All Mods)[/h1]

Mod Barony with plain files. No C++, no compiler, no engine knowledge. Write a small .json, drop it in your mods folder, and the game loads it as a real class, item, monster, spell or status effect. Add a short script next to it if you want it to actually do something.

[b]S.A.M is a dependency, not a playable mod.[/b] On its own it adds nothing to play. It is the thing other mods are built on. Install it once and every S.A.M mod works.

[b]READ THIS: subscribing here does NOT install it.[/b] S.A.M is a modified barony.exe, and Steam cannot replace a game's .exe by itself. You have to run the installer one time. Full steps are right below. It takes about a minute.

[hr][/hr]

[h1]HOW TO INSTALL (step by step)[/h1]

[b]Step 1.[/b] Own Barony on Steam and launch the game once, so its files exist. Then close the game.

[b]Step 2.[/b] Click this link to open the download page:
[url=https://github.com/spider12223/SAM-Framework/releases/latest]https://github.com/spider12223/SAM-Framework/releases/latest[/url]

[b]Step 3.[/b] On that page, scroll down to the [b]Assets[/b] list. Click the file called:
[b]SAM_Framework_Installer.exe[/b]
It downloads to your Downloads folder. There is only one installer and it never goes out of date.

[b]Step 4.[/b] Open your [b]Downloads[/b] folder and double click [b]SAM_Framework_Installer.exe[/b].

[b]Step 5.[/b] If Windows shows a blue box saying "Windows protected your PC", click [b]More info[/b], then click [b]Run anyway[/b]. The installer is not code signed, which is the only reason Windows warns.

[b]Step 6.[/b] The installer finds your Barony folder on its own. Check that the path it shows is your Barony install, then click [b]Install[/b].

[b]Step 7.[/b] Let it finish. It saves your original game file as [b]barony_vanilla.exe[/b] (that is your backup), then puts the S.A.M version in place. Close the installer.

[b]Step 8.[/b] Launch Barony from Steam like normal.

[b]Step 9.[/b] Look at the bottom of the main menu. If you see a shield icon and the words [b]S.A.M Framework[/b], you are done. If you do not see it, S.A.M did not install, so repeat steps 3 to 7.

[b]You only do this once.[/b] To update later, just run the installer again. It always grabs the newest version.

[hr][/hr]

[h1]HOW TO USE A MOD[/h1]

[b]Step 1.[/b] Get a S.A.M mod. It is a folder. If you got a .zip, unzip it first.

[b]Step 2.[/b] Put that folder inside your Barony mods folder. The usual path is:
[b]C:\Program Files (x86)\Steam\steamapps\common\Barony\mods\[/b]

[b]Step 3.[/b] Launch Barony, click [b]Mods[/b] on the main menu, click the [b]Local Mods[/b] tab, then click [b]Load Mod[/b] next to your mod.

[b]Step 4.[/b] Click [b]Start Modded Game[/b] at the bottom. Your class or item is now in the game.

[b]Using a Workshop mod instead?[/b] Subscribe to it, then let Steam finish downloading before you launch. If a Workshop mod's [b]Load Mod[/b] button does nothing, it means Steam has not downloaded the files yet. Fully close Steam, reopen it, wait for the download to finish, then launch.

[hr][/hr]

[h1]HOW TO GO BACK TO NORMAL BARONY[/h1]

Delete barony.exe, then rename [b]barony_vanilla.exe[/b] back to [b]barony.exe[/b]. Or right click Barony in Steam, pick Properties, then Installed Files, then Verify integrity of game files.

Nothing else is ever changed. S.A.M does not edit any of the game's own files, and it backs up your saves while mods are active.

[h1]WHAT YOU CAN MAKE[/h1]
[list]
[*][b]Classes.[/b] Attributes, skills, starting gear (including your own custom items), starting spells, level up growth, starting gold, and a character select portrait.
[*][b]Items and weapons.[/b] Real entries in Barony's item table, with category, equip slot, weight, value, ATK and AC, your own model and icon.
[*][b]Monsters.[/b] Variants of any creature with your own stats, gear, name, followers and spawn floors.
[*][b]Spells.[/b] Custom spells that really cast, through the game's own projectile pipeline.
[*][b]Status effects.[/b] Your own named buffs and afflictions, with duration, strength, AC and damage changes.
[*][b]Playable races.[/b] Play as a monster body.
[*][b]Sounds.[/b] Ship your own .ogg files and play them from a script.
[*][b]Custom .vox models.[/b] Your own voxel models, loaded at startup.
[*][b]Tinkering recipes.[/b] Add your own items to the tinkering kit's crafting grid, re-price a vanilla recipe, or hide one to make room.
[*][b]Vanilla patches.[/b] Edit the game's data in layers, so two mods stack instead of overwriting each other.
[/list]

[h1]SCRIPTING (Lua, JavaScript, TypeScript)[/h1]

JSON says what your content [i]is[/i]. A script says what it [i]does[/i]. Drop a .lua, .js or .ts file next to any class or item JSON and it loads automatically. TypeScript is compiled for you. Still no C++ and no compiler.

[b]53 events[/b] to react to. Combat (on_hit, on_kill, on_block, on_before_damage), lifecycle (level up, death, floor change, game start), items (equip, use, pick up off the ground, identify, break), shops, chests, traps, monsters, key presses, a per tick handler, world things (doors, levers, fountains, sinks, orb pedestals, teleporters, projectile hits), and bound actions (Attack, Defend, Use) that follow whatever the player rebound them to.

[b]91 functions[/b] to act with. Grant items and gold (blessed, cursed, any quality), read and set stats, level a player up for real, apply and read status effects, cast spells (forward, at a target, or at a spot), deal or change damage, move speed, teleport, spawn monsters and items, full monster control by id, timers, saved data that survives between runs, screen effects, and live patching of existing content that reverts when your mod unloads.

Only two functions ever get called: [b]on_event(event)[/b] and [b]on_tick(event)[/b]. Everything else arrives as [b]event.name[/b] inside on_event.

[b]Safe by design.[/b] Scripts run sandboxed with a memory cap and an instruction limit. A broken script switches itself off and writes why to the log. It cannot crash your game.

[h1]MAKE A MOD IN YOUR BROWSER[/h1]

The easiest way. Nothing to download:

[b][url=https://spider12223.github.io/SAM-Framework/]S.A.M Mod Builder[/url][/b]

Build a class, item, monster, spell or effect with dropdowns and sliders, watch the JSON update live, then export a ready to use .zip. It checks your work against the same rules the game uses, so if it exports, it loads. You can also import an existing mod to edit it.

[h1]PUBLISHING YOUR OWN MOD[/h1]
[list]
[*][b]1.[/b] Build and test it locally with S.A.M installed.
[*][b]2.[/b] Put a [b]preview.png[/b] in your mod folder. That exact filename becomes the thumbnail, and the upload can fail without one.
[*][b]3.[/b] Upload the folder as a Workshop item.
[*][b]4.[/b] Add [b]S.A.M Framework[/b] as a Required Item on your Workshop page, and say in your description that players must run the S.A.M installer. Subscribing alone does not install the exe.
[/list]

Before that: custom items show their real damage and AC in tooltips, custom classes show their own portrait everywhere, you can put your own items into a class's starting gear from the Mod Builder, and scripts gained screen effects for big moments (a flash, an anime style impact burst, camera shake, and a brief freeze frame). Earlier still: custom status effects, custom spells with real casting, playable monster races, custom sounds, floating companions, and a visual block builder that writes the script for you.
