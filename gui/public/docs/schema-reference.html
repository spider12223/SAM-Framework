mod.json — Mod Manifest

Declares a mod's identity, its S.A.M/Barony version requirements, and the class/item/patch/plugin files it ships.

patch.json — Layered Patch

Additively edits an existing data file (e.g. items/items.json) without replacing it, so multiple mods stack instead of conflicting.

Required: target, operations

FieldTypeDescription
$schemaoptionalstringOptional. URL of this JSON Schema, so editors give autocomplete + validation as you type. Ignored by S.A.M itself.
targetrequiredstringLogical path of the data file to patch, as Barony reads it, e.g. 'items/items.json' or 'data/gameplaymodifiers.json'. Must be a file Barony loads via its data-file system.
operationsrequiredarrayOperations applied in order to the target's JSON. Later operations see the result of earlier ones (and of earlier-loaded mods).
each array entry:
FieldTypeDescription
oprequiredenumedit_field: set an EXISTING field to 'value' (never replaces a whole entry; a missing path is warned + skipped, catching typos). add_entry: set/create the field at 'path' to 'value' (adds a new key). remove_field: delete the field at 'path'. multiply_field: multiply the numeric field at 'path' by the numeric 'value' (integer fields stay integers) — handy for balance mods.
edit_fieldadd_entryremove_fieldmultiply_field
pathrequiredstringDot-separated path into the target JSON, matching the file's real structure. For items/items.json the item keys are LOWERCASE internal names and fields use snake_case, e.g. 'items.iron_sword.gold_value' or 'items.wooden_shield.weight_value'. (Keys are looked up literally; use the names exactly as they appear in the file.)
minLen 1
valueoptional—The new value (edit_field/add_entry) or the numeric factor (multiply_field). Omitted for remove_field. May be any JSON type for edit_field/add_entry; must be a number for multiply_field.
